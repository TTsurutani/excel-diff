## 確定済み設計方針

| 項目 | 決定内容 | 補足 |
|---|---|---|
| UIライブラリ | `tkinter`（Python標準） | exe同梱可、追加依存なし |
| ビルド成果物 | `excel-diff-gui.exe`（CLIと**別exe**） | `build_gui.bat` で生成 |
| ウィンドウ | 最小サイズ `700×640`、リサイズ可 | `minsize` 指定 + `weight` でリサイズ追従 |
| 実行方式 | バックグラウンドスレッド＋キューポーリング | `threading.Thread` + `queue.Queue` |
| 設定永続化 | `gui_settings.json`（exe横に配置） | タブごとの最終入力値を保存 |
| パターン管理 | GUI完結でフル実装（ウィザード形式） | CLIを知らない利用者でも使える |

## タブ① ファイル比較

### 対応CLIコマンド
```
excel-diff <old> <new> [-o PATH] [--diff-mode key --key-cols C] [--sheet] [--include-cols] [--matchers] [--strikethrough] [--open]
```

### UI レイアウト
```
旧ファイル  [________________________________] [参照]
新ファイル  [________________________________] [参照]

─── 差分モード ──────────────────────────────────────
  ● LCS（行の出現順で比較・デフォルト）
  ○ キーJOIN（キー列の値で行を対応付ける）
       キー列  [____________]  例: C  または  B,C

▼ オプション（折りたたみ）
  出力HTML      [________________________________] [参照]
  比較シート    [____________]  ※空=全シート
  比較列        [____________]  例: A:C,E
  マッチャーJSON[________________________________] [参照]
  ☐ 取り消し線も差分として扱う
  ☐ 完了後ブラウザで開く

                              [  実行  ]
```

### 動作仕様
- 旧・新ファイルが未指定のまま実行するとエラーメッセージ表示（実行しない）
- 「LCS」選択中はキー列入力欄をグレーアウト
- 「キーJOIN」選択中はキー列入力欄が活性化。未入力のまま実行しようとするとエラー
- バックグラウンドで `diff_files` + `render` を実行
- 完了後、ログにクリック可能な出力ファイルパスを表示


### UI レイアウト
```
ブックファイル [________________________________] [参照]
前置文字列     [____________]
後置文字列     [____________]
出力先フォルダ [________________________________] [参照]
               ※空=ブックと同じフォルダ

出力プレビュー:
  ┌──────────────┬─────────────────────────┐
  │ シート名      │ 出力ファイル名           │
  ├──────────────┼─────────────────────────┤
  │ 売上         │ PRE_売上_SUF.xlsx        │
```

### 動作仕様
- ブックファイル選択時にシート名を読み込み、プレビューテーブルを即時生成

- シート名にファイル名不正文字が含まれ `_` に置換される場合、プレビューで赤字表示
- バックグラウンドで `split_workbook` を実行

- ウィザードはステップ②のまま（比較は実行しない）

```
テンプレート [日付8桁（_20240101.xlsx）▼]

    日付6桁（例: report_202401.xlsx）    →  ^(.+?)_\d{6}\.xlsx$
    バージョン番号（例: report_v2.xlsx） →  ^(.+?)_v\d+\.xlsx$
    連番（例: report_001.xlsx）          →  ^(.+?)_\d+\.xlsx$

│ inv_20240101.xlsx  │ inv_20240201.xlsx   │ inv      │  ✓   │
│ report_old.xlsx    │ -                  │(マッチなし)│  -   │
└────────────────────┴────────────────────┴──────────┴──────┘
  ※ キーが不一致の行: 黄色強調
  ※ キー衝突の行: 赤色強調

─── 検証結果 ─────────────────────────────────────────────────────────
（[検証実行] 後に表示。検証OK後に [保存] が活性化）

[← 戻る]            [検証実行]          [保存]
```

**検証エラーの日本語表示**

| エラー種別 | 表示内容 |
|---|---|
| `invalid_regex` | 正規表現の文法エラー。括弧の対応や記号のエスケープを確認してください。 |
| `no_match` | マッチしないファイルがあります。ファイル名のパターンと正規表現が合っていません。テンプレートを変えて試してください。 |
| `key_mismatch` | 旧と新でキーが一致しないペアがあります。キャプチャ範囲が変動部分（日付等）を含んでいる可能性があります。 |
| `key_collision` | 同一キーに複数ファイルがマッチしています。正規表現が広すぎます。 |

## バックグラウンド実行の設計

```
メインスレッド（UI）
  ↓ 実行ボタン押下
  → Thread(target=_worker) 起動
  → ボタン無効化、ログに「実行中...」表示
  → after(100, _poll_queue) でキューを定期ポーリング

ワーカースレッド
  → 処理を実行
  → 進捗を queue.put({"log": "メッセージ"}) で送信
  → 完了/エラーを queue.put({"done": True, "error": None}) で通知

_poll_queue
  → キューからメッセージを取り出してログに追記
  → done 受信でボタン再有効化、after ループ終了
```
